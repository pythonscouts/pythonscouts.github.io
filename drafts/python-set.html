Estructuras de datos en Python: Conjuntos (sets)

Leodanis Pozo Ramos
23 abr 2026

Los conjuntos o sets almacenan colecciones de valores únicos sin orden definido. Son muy útiles para pruebas de pertenencia y operaciones matemáticas.

En este artículo, aprenderás cómo implementar un conjunto en Python, incluyendo sus operaciones y características principales.

¿Qué es un conjunto o set?

Un conjunto es un contenedor que almacena una colección de valores únicos sin un orden particular. Los conjuntos implementan las mismas operaciones que sus equivalentes matemáticos. Son especialmente útiles para pruebas de pertenencia, donde necesitas saber si un valor está o no en el contenedor.

Las características fundamentales de un conjunto incluyen las siguientes:

  • Unicidad: No permite elementos duplicados. Si intentas añadir un valor que ya existe, se ignora.
  • Sin orden definido: Los elementos no tienen una posición específica.
  • Valores hashables: Solo acepta elementos que sean hashables, lo que garantiza la unicidad.

En Python, los conjuntos están integrados como el tipo set, pero en este artículo implementarás tu propio conjunto usando una lista como estructura subyacente para comprender su funcionamiento interno.

Operaciones comunes en un conjunto

A continuación, un resumen de las operaciones que soportará tu conjunto:

Operación Descripción
s = Set() Construye un conjunto vacío.
s = Set(iterable) Construye un conjunto con elementos únicos de iterable.
s.add(element) Añade element al conjunto.
s.remove(element) Elimina element del conjunto. Lanza KeyError si no existe.
s.discard(element) Elimina element del conjunto si está presente. No lanza error si no existe.
s.pop() Extrae un elemento del conjunto.
s.clear() Elimina todos los elementos del conjunto.
s.update(other) Actualiza el conjunto con los elementos de other.
s.is_subset(other) Devuelve True si el conjunto es subconjunto de other.
s.is_superset(other) Devuelve True si el conjunto es superconjunto de other.
s.is_disjoint(other) Devuelve True si el conjunto no tiene elementos en común con other.
s.union(other) Devuelve un nuevo conjunto con la unión de ambos.
s.intersection(other) Devuelve un nuevo conjunto con la intersección de ambos.
s.difference(other) Devuelve un nuevo conjunto con la diferencia entre ambos.
s == other Devuelve True si ambos conjuntos contienen los mismos elementos.
len(s) Devuelve el número de elementos en el conjunto.
element in s Devuelve True si element está en el conjunto, False en caso contrario.

Implementación de un conjunto en Python

A continuación encontrarás la implementación del conjunto en Python:

Python - set_adt.py
from typing import Any, Iterator, Optional, Sequence


class Set:
    """Implement a Set abstract data type."""

    def __init__(self, iterable: Optional[Sequence[Any]] = None, /) -> None:
        self._data: list[Any] = []
        if iterable is not None and len(iterable) > 0:
            for element in iterable:
                self.__add(element)

    def add(self, element: Any) -> None:
        """Add element to set."""
        hash(element)
        if element not in self._data:
            self._data.append(element)

    __add = add  # Avoid breaking the class by a subclasser

    def remove(self, element: Any) -> None:
        """Remove an element from set."""
        try:
            self._data.remove(element)
        except ValueError:
            raise KeyError(f"{element} not in set") from None

    def discard(self, element: Any) -> None:
        """Remove element from set if present."""
        try:
            self._data.remove(element)
        except ValueError:
            pass

    def pop(self) -> Any:
        """Pop an element from set."""
        try:
            return self._data.pop()
        except IndexError:
            raise KeyError("pop from an empty set") from None

    def clear(self) -> None:
        """Remove all the elements from set."""
        self._data.clear()

    def update(self, other: "Set") -> None:
        """Update set with elements from other."""
        self._validate_other(other)
        for element in other:
            self.__add(element)

    def _validate_other(self, other: "Set") -> None:
        if other.__class__ is not self.__class__:
            raise TypeError("Set object expected")

    def is_subset(self, other: "Set") -> bool:
        """Return True if set is subset of other, False otherwise."""
        self._validate_other(other)
        for element in self:
            if element not in other:
                return False
        return True

    def is_superset(self, other: "Set") -> bool:
        """Return True if set is superset of other, False otherwise."""
        self._validate_other(other)
        for element in other:
            if element not in self:
                return False
        return True

    def is_disjoint(self, other: "Set") -> bool:
        """Return True if set has no elements in common with other."""
        self._validate_other(other)
        for element in self:
            if element in other:
                return False
        return True

    def union(self, other: "Set") -> "Set":
        """Return a new set that is the union of set and other."""
        self._validate_other(other)
        new_set = type(self)()
        new_set._data.extend(self._data)
        for element in other:
            new_set.__add(element)
        return new_set

    def intersection(self, other: "Set") -> "Set":
        """Return a new set that is the intersection of set with other."""
        self._validate_other(other)
        new_set = type(self)()
        for element in self:
            if element in other:
                new_set.__add(element)
        return new_set

    def difference(self, other: "Set") -> "Set":
        """Return a new set with the difference between set and other."""
        self._validate_other(other)
        new_set = type(self)()
        for element in self:
            if element not in other:
                new_set.__add(element)
        return new_set

    def __eq__(self, other: "Set") -> bool:
        self._validate_other(other)
        return sorted(self._data) == sorted(other._data)

    def __len__(self) -> int:
        return len(self._data)

    def __contains__(self, element: Any) -> bool:
        return element in self._data

    def __iter__(self) -> Iterator:
        yield from self._data

    def __repr__(self) -> str:
        return f"{self.__class__.__name__}({self._data})"

    __str__ = __repr__

El atributo ._data es una lista que almacena los elementos del conjunto. Aunque esta implementación usa una lista internamente, garantiza la unicidad de los elementos al verificar su presencia antes de añadir nuevos valores.

Los métodos principales te permiten ofrecer las siguientes funcionalidades:

  • .add(): añade un elemento al conjunto solo si no existe previamente. Verifica que el elemento sea hashable llamando a hash().
  • .remove(): elimina un elemento del conjunto. Lanza KeyError si el elemento no existe.
  • .discard(): elimina un elemento si está presente, sin lanzar error si no existe.
  • .pop(): extrae un elemento del conjunto. Lanza KeyError si el conjunto está vacío.
  • .clear(): elimina todos los elementos.
  • .update(): actualiza el conjunto con los elementos de otro conjunto.

Las operaciones de conjuntos matemáticos son las siguientes:

  • .is_subset(): comprueba si todos los elementos están contenidos en otro conjunto.
  • .is_superset(): comprueba si el conjunto contiene todos los elementos de otro.
  • .is_disjoint(): comprueba si no hay elementos en común con otro conjunto.
  • .union(): devuelve un nuevo conjunto con todos los elementos de ambos.
  • .intersection(): devuelve un nuevo conjunto con los elementos comunes.
  • .difference(): devuelve un nuevo conjunto con los elementos que no están en el otro.

Los métodos especiales te permiten ofrecer las siguientes funcionalidades:

  • .__eq__(): compara dos conjuntos por igualdad, ordenando los elementos para comparar.
  • .__len__(): devuelve el número de elementos en el conjunto.
  • .__contains__(): comprueba si un elemento está en el conjunto.
  • .__iter__(): permite iterar sobre los elementos del conjunto.
  • .__repr__() y .__str__(): devuelven representaciones de cadena legibles.

Ejemplo de uso del conjunto

En esta sección verás ejemplos prácticos de cómo crear y manipular instancias de tu clase Set.

Inicialización

Para crear un Set, puedes hacerlo sin argumentos o pasando un iterable. Los duplicados se eliminan automáticamente:

Python REPL
>>> numbers = Set()
>>> numbers
Set([])

>>> numbers = Set([1, 2, 3, 3, 3])
>>> numbers
Set([1, 2, 3])

Añadir y eliminar elementos

Puedes añadir elementos y eliminarlos por su valor:

Python REPL
>>> numbers = Set()
>>> numbers.add(1)
>>> numbers.add(2)
>>> numbers.add(2)  # Duplicado: se ignora
>>> numbers
Set([1, 2])

>>> numbers.remove(1)
>>> numbers
Set([2])

Si intentas añadir un elemento que no sea hashable, obtendrás un error:

Python REPL
>>> numbers = Set()
>>> numbers.add([1, 2])
Traceback (most recent call last):
    ...
TypeError: unhashable type: 'list'

Operaciones de conjuntos

Puedes realizar las operaciones clásicas de conjuntos matemáticos:

Python REPL
>>> set_a = Set([1, 2, 3])
>>> set_b = Set([2, 3, 4, 5])

>>> set_a.union(set_b)
Set([1, 2, 3, 4, 5])

>>> set_a.intersection(set_b)
Set([2, 3])

>>> set_a.difference(set_b)
Set([1])

Relaciones entre conjuntos

Puedes verificar relaciones como subconjunto, superconjunto y disyunción:

Python REPL
>>> set_a = Set([1, 2])
>>> set_b = Set([1, 2, 3, 4])

>>> set_a.is_subset(set_b)
True
>>> set_b.is_superset(set_a)
True

>>> set_c = Set([5, 6])
>>> set_a.is_disjoint(set_c)
True

Longitud y pertenencia

Puedes consultar la longitud del conjunto y verificar si un elemento está contenido en él:

Python REPL
>>> numbers = Set([10, 20, 30])

>>> len(numbers)
3

>>> 20 in numbers
True
>>> 99 in numbers
False

Iteración

El método .__iter__() permite iterar sobre los elementos del conjunto:

Python REPL
>>> numbers = Set([1, 2, 3])

>>> for element in numbers:
...     print(element)
...
1
2
3

Conclusión

Este conocimiento te permite comprender cómo funcionan internamente los conjuntos y cuándo son la estructura más adecuada para resolver problemas que involucren colecciones de valores únicos y pruebas de pertenencia.
